# Add Browser Viewer For Combined Terrain Mesh

Status: executed

## Summary

Add a non-Gazebo viewer stage at the end of the pipeline. The app will always
generate a combined terrain `GLB` plus a browser viewer under
`outputs/<world-name>/viewer/`, then print a command for opening it locally.
Gazebo remains unchanged.

## Key Changes

- Add runtime dependencies: `trimesh` and `pygltflib`.
- Add a viewer generation module that reads the DEM and `tiles.csv`, rebuilds
  each tile mesh using existing mesh logic, applies tile center offsets, and
  exports one combined `terrain.glb`.
- Add height-based vertex colors to make the terrain readable without Gazebo.
- Generate `viewer/index.html` using Three.js from CDN.
- Run viewer generation after Gazebo generation in the main pipeline.
- Add `gz-terrain-gen-viewer` to serve the generated viewer locally.
- Update metadata with viewer directory, GLB path, HTML path, vertex count, and
  face count.

## Documentation

- Update README and architecture docs with viewer output and usage.
- Clarify that the viewer is for inspecting the combined terrain mesh only.
- Note that the viewer uses Three.js from CDN.

## Assumptions

- Browser viewer is preferred over a desktop Python viewer.
- GLB/GLTF is the preferred combined mesh format.
- Viewer artifacts should always be generated by the normal pipeline.
- The main pipeline should not automatically open a browser.
- Three.js CDN usage is acceptable for the first viewer version.
- The viewer does not replace Gazebo.
