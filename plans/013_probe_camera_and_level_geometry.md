# Place Level Probe And GUI Camera Above First Terrain Tile

Status: executed

## Summary

Update Gazebo world generation so `level_probe` starts above the first terrain
tile surface, and the GUI camera looks straight down from above the same point.
Make level geometry Z size configurable from the CLI with default `1500 m`,
while level X/Y geometry follows each tile's actual generated size.

## Key Changes

- Read the first tile mesh and compute the terrain elevation at local center
  `(0, 0)`, falling back to the nearest center vertex.
- Place `level_probe` at the first tile center plus 30 m.
- Place the GUI camera at the first tile center plus 100 m with top-down pitch.
- Set the internal `level_probe` model pose to zero so world placement controls
  the full pose.
- Add `--level-z-size-m`, defaulting to `1500`.
- Generate level geometry X/Y from tile size and Z from `--level-z-size-m`.
- Record probe pose, GUI camera pose, and level Z size in Gazebo metadata.

## Assumptions

- "First terrain" means the first tile listed in `tiles.csv`.
- Probe clearance is fixed at 30 m.
- GUI camera clearance is fixed at 100 m.
- Level X/Y geometry should match tile size while the Gazebo level buffer
  remains unchanged.
- Generated files under `outputs/` are regenerated by the pipeline.
